# Discovery Inbox

Drop raw idea documents, mockups, screenshots, transcripts, links, or any other
starting materials in this folder.

Once materials land here, the next step is the `discovery-to-composition`
workflow in the harness (`.harness/platform/workflow/discovery-to-composition.md`):
read the inputs, distill them into a problem statement, then choose the matching
harness composition from `.harness/platform/compositions/`.

This folder is for raw inputs only. Distilled artifacts (problem statement,
personas, MVP scope, requirements) move up into `docs/discovery/` and
`docs/product/` once authored from templates in
`.harness/platform/templates/`.
